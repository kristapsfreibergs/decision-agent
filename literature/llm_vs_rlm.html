Large Language Models
versus Recursive Language Models

A comparative specification of how standard transformer-based LLMs and the emerging Recursive Language Model paradigm process language — covering architecture, inference mechanics, complexity, and research lineage.

Fig. 1 Autoregressive Transformer pipeline. Input tokens are embedded and positionally encoded, then processed by N stacked transformer blocks. Each block applies multi-head self-attention — where every token attends to every other (O(n²) complexity) — followed by a feed-forward sublayer. Output is produced autoregressively, one token per forward pass. State-of-the-art extensions replace components: Sparse MoE scales capacity (Fedus et al., 2022), RLHF aligns outputs to human preference (Ouyang et al., 2022), and Mamba substitutes attention with linear-time SSM (Gu & Dao, 2023).

Fig. 2 RLM Recursive Inference Loop. Rather than loading all context into a fixed attention window, the RLM treats the long document as an external variable in a REPL (Read-Eval-Print Loop). At each step the underlying LLM generates code — search, slice, filter, or recursive calls — to retrieve only the relevant sub-context. Execution results are fed back into the next iteration. This recursive decomposition continues until the sub-task is resolved, after which partial results are merged hierarchically. Zhang et al. (2025) demonstrate 100× context window expansion at a fraction of standard attention cost.

Fig 4 — Property Comparison

Fig. 4 LLM vs RLM — Key Properties
Property LLM — Transformer RLM — Recursive Inference
Context handling Fixed window — all tokens loaded into attention (4K–200K typical). Cost grows O(n²) with length. Unbounded — document stored externally; code retrieves only relevant slices. 100× expansion demonstrated.
Processing depth Fixed — always N layers regardless of input complexity. Adaptive — recursion depth scales with task complexity; simple queries terminate early.
Attention complexity O(n²) per layer — quadratic in sequence length. Mitigated by flash attention, sparse attention. Sub-linear overall — each recursive call attends over a small slice; total cost ≪ full-context attention.
Compositionality Emergent via scale — no explicit recursive structure. Explicit — recursive decomposition mirrors hierarchical language structure (Chomsky, 1957).
Parallelism Fully parallel — all tokens processed simultaneously during training and prefill. Sequential by design — each recursive call depends on the previous result.
Training requirement Requires training / fine-tuning from scratch or SFT. Training-free — inference-time strategy applied to any existing LLM.
Failure modes Hallucination over long context; lost-in-the-middle; context drift. Recursive loop divergence; code execution errors; sub-result aggregation errors.
Best suited for Short-to-medium context tasks; generation; structured output; low-latency inference. Very long documents; multi-hop retrieval; tasks requiring explicit hierarchical reasoning.
Key reference Vaswani et al., NeurIPS 2017 Zhang, Kraska, Khattab — arXiv 2512.24601, 2025
Fig. 4 LLM vs RLM property comparison. The two paradigms are complementary rather than competing: RLM is an inference strategy that wraps a standard LLM, adding recursive decomposition on top of its existing capabilities. The primary trade-off is parallelism (LLM strength) versus unbounded context and adaptive depth (RLM strength).

References

  • Vaswani et al.
    NeurIPS 2017     Attention Is All You Need. Introduced the Transformer architecture — multi-head self-attention, positional encoding, encoder-decoder stack. Foundation of all modern LLMs.
  • Socher et al.
    EMNLP 2013     Recursive Deep Models for Semantic Compositionality Over a Sentiment Treebank. Recursive Neural Tensor Networks exploiting parse-tree structure for compositional meaning — early foundation of recursive NLP.
  • Dehghani et al.
    ICLR 2019     Universal Transformers. Transformer variant with recurrent depth and Adaptive Computation Time (ACT). Bridges fixed-depth transformers and recurrent networks; Turing-complete under certain assumptions.
  • Brown et al.
    NeurIPS 2020     Language Models are Few-Shot Learners. GPT-3 — 175B parameter autoregressive LLM demonstrating emergent few-shot capabilities from scale alone.
  • Ouyang et al.
    NeurIPS 2022     Training Language Models to Follow Instructions with Human Feedback. InstructGPT — RLHF alignment pipeline. Established the SFT → RM → PPO recipe adopted by GPT-4, Claude, Gemini.
  • Fedus et al.
    JMLR 2022     Switch Transformers: Scaling to Trillion Parameter Models with Simple and Efficient Sparsity. Sparse Mixture-of-Experts replacing dense FFN layers. Routes each token to one expert, scaling capacity without proportional compute increase.
  • Giannou et al.
    ICML 2023     Looped Transformers as Programmable Computers. Demonstrates that a looped transformer (same weights applied iteratively) can emulate arbitrary programs — linear algebra, in-context learning, calculator. Establishes theoretical basis for recursive transformer computation.
  • Gu & Dao
    arXiv 2023     Mamba: Linear-Time Sequence Modeling with Selective State Spaces. State Space Model achieving O(n) inference complexity vs O(n²) attention. Selective state updates conditioned on input. 5× faster inference than transformers at equivalent perplexity.
  • Zhang, Kraska,
    Khattab · MIT
    arXiv 2025     Recursive Language Models. arXiv:2512.24601. Introduces RLM — an inference paradigm treating long prompts as external variables in a REPL. The LLM recursively decomposes tasks and examines context via generated code (search, slice, filter, recurse), achieving 100×+ effective context expansion at a fraction of full-attention cost.
