Reference Architecture Gap Map - Current vs Target

Back to index

Two views are used here with the same principle as reference agentic AI architectures: the capability map shows what the system must be able to do; the technology map shows the concrete components that implement those capabilities.

Memory is shown as a separate multi-layer system, not one generic store: working/context memory, run episodic memory, evidence memory, semantic/vector memory, ontology memory, and user/org profile memory.

1. Layered Capability Gap Map

This is the most important architecture diagram. It follows the reference structure: user/client entry, orchestration/control plane, specialized agents, tools/integrations, memory/knowledge, monitoring, reliability, governance, and foundation capabilities. Each node says what exists today and what missing functionality impairs.

This is a direct tree view: left side is what the repository has now, right side is the best-practice target shape for a long-running autonomous decision-agent platform. Colors use the same gap-map legend.

    • .tmp/ scratchCurrent: Temporary working files and generated scratch artifacts.

      • src/decision_agent/ Python package Current: Main Python package where API, modules, provider abstractions, memory, and audit live.
