Run Lifecycle Sequence

End-to-end flow from task submission through architecture selection, contract generation, worker execution, governance checks, and audit recording. State transitions are derived by replaying audit.jsonl — see 17 — C4 & State Machines for the full state machine. 

sequenceDiagram
  autonumber
  actor User
  participant UI as UI or CLI
  participant Server as server.py
  participant Runs as runs.service
  participant Router as decision router
  participant Arch as architecture modules
  participant Contracts as contracts modules
  participant Sched as scheduler
  participant Worker as worker runner
  participant Provider as model provider
  participant Gov as governance (DSC/PAAP/DAR)
  participant Data as data/runs

  User->>UI: Submit task
  UI->>Server: POST /api/runs
  Server->>Runs: create_run(task)
  Runs->>Router: classify_decision_type(task)
  Router-->>Runs: decision_type
  Runs->>Data: task.json, run-record.json, audit.jsonl

  alt Static architecture exists
    Runs->>Arch: find_architecture_for_decision()
    Arch-->>Runs: architecture definition
    Runs->>Contracts: instantiate and validate contracts
    Contracts-->>Data: contracts/*.json
  else Dynamic architecture
    UI->>Server: POST /architecture/build
    Server->>Runs: build_architecture_proposal()
    Runs->>Arch: classify goal, build topology, decompose packages
    Arch-->>Data: planning-artifact.json, architecture-proposal.json
    User->>UI: Review and approve architecture
    UI->>Server: POST /architecture/approve
    UI->>Server: POST /architecture/generate-contracts
    Runs->>Contracts: generate contracts from proposal
    Contracts-->>Data: generated-contracts/*.json
  end

  UI->>Server: POST /start
  Server->>Data: append run_started
  UI->>Server: POST /schedule
  Server->>Sched: get ready workers by deps + phase gates
  loop Ready workers
    Sched->>Worker: run_worker(contract)
    Worker->>Provider: complete_with_tools(system, messages, tools)
    Provider-->>Worker: tool_use or final JSON
    Worker->>Worker: execute bounded tools (read/write/query_sql/memory_search)
    Worker->>Contracts: schema + contractual output validation
    Contracts->>Gov: PAAP evidence scoring + DSC scope check
    Worker->>Gov: DAR receipt for proposed action
    Worker->>Data: outputs/*.json, evidence/*.json, authorization/*.json, audit
  end

  alt Human / phase gate required
    User->>UI: Approve or reject gate
    UI->>Server: POST gate endpoint
    Server->>Data: append gate_approved / gate_rejected
  end

  UI->>Server: GET /api/dashboard
  Server-->>UI: enriched run state (derived from audit.jsonl replay)

Runtime Artifact Model

Every run writes its complete state to data/runs/{run_id}/. The audit log is the source of truth — all other files are derived outputs. external_inputs/ is planned V1 for long-running pause/resume. 

flowchart LR

  Task["task.json\noriginal task input"]
  Record["run-record.json\nlayer_config · provider · context"]
  Audit["audit.jsonl\nappend-only event log\nsource of truth"]

  Contracts["contracts/*.json\nstatic architecture runs"]
  Proposal["architecture-proposal.json\ndynamic runs"]
  Planning["planning-artifact.json"]
  Generated["generated-contracts/*.json"]

  Outputs["outputs/*.json\nworker structured output"]
  Scope["scope.json\nDSC scope contract"]
  Evidence["evidence/*.json\nPAAP scored records"]
  Auth["authorization/*.json\nDAR receipts"]
  Checkpoints["checkpoints/*.json\nworker state after each tool call"]
  External["external_inputs/*.json\nplanned V1 — long-running pause"]
