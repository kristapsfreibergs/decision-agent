Decision Agent Logical Architecture

A compact explanation of the project: a procurement decision is processed by a sequence of agents. Each agent is built from atomic operators, patches shared decision state, and is checked by governance layers.

One core idea The system turns a procurement task into a controlled decision pipeline.
Reusable building blocks Agents are composed from atomic operators such as extract, score, rank, and mem_write.
Control layer Governance checks evidence scope, evidence quality, authorization, and valid state transitions.

Procurement Decision Flow

The procurement decision graph has nine agents executed in sequence.

  • RequirementAgentextracts and structures what's being procured
  • EvidenceAgentresearches market, retrieves prior evidence from memory
  • NormalizationAgentstandardizes currencies, units, dates
  • EligibilityAgenteliminates vendors failing hard constraints
  • EvaluationAgentscores and ranks surviving vendors
  • ComparisonAgentcompares top vendors side by side
  • RecommendationAgentproduces the decision brief, persists to memory
  • StateUpdateAgentfinalizes state, persists rankings to memory
  • StateValidationAgentvalidates invariants, records outcome

What Agents Are Made Of

Agents are not monolithic. They are short chains of atomic operators. The EvidenceAgent is the clearest example because it combines memory, LLM extraction/classification, deterministic validation, state update, and audit logging.

Atomic Operator Families

Operators are the smallest reusable primitives. LLM operators handle language tasks. Deterministic operators handle rules, thresholds, scoring, state changes, and audit. Memory operators connect the current run to past runs.

Shared State Lifecycle

Agents do not mutate shared state directly. Each successful operator returns a state patch. The state machine controls which decision phases are valid. 

stateDiagram-v2
  [*] --> Draft
  Draft --> EvidenceIncomplete
  EvidenceIncomplete --> Eligible
  Eligible --> Evaluated
  Evaluated --> Recommended
  Recommended --> Approved
  Approved --> StateUpdated
  StateUpdated --> Validated
  Validated --> [*]

Governance And Memory

Governance is shown as an overlay rather than as another agent. It validates whether evidence is in scope, whether evidence is strong enough, whether proposed actions are authorized, and whether humans must approve.
